Agent as Policies: LLM Powered Robot Agent for Long-Horizon Tasks

Anonymous

Abstract

We introduce Agent as Policies (AgP), a Large Language Model (LLM)-powered robot agent that is efficacy in long-horizon tasks, adapts to novel tasks, and resolves complex task exploration without human intervention. AgP consists of four key components: 1) a planner that decomposes user queries into executable sub-plans, 2) a human-inspired memory bank that retains both long-term and short-term memory for skill enhancement and planning error reduction, 3) a skill library that provides necessary tools for plans, and 4) an execution module that serves as system low-level foundation and provides planner with necessary feedback. AgP interacts with GPT-4 via black-box queries, eliminating the need for model parameter fine-tuning. The actions developed by AgP are temporally extended, interpretable, and compositional, which rapidly enhances the agent’s abilities and mitigates catastrophic forgetting. Together with a skill library, AgP exhibits exceptional proficiency in robotics on long-horizon tasks. It shows robust adaptability when encountering novel environments and evolves by incorporating new skills. In real robot experiments, our method showcases a strong capability to handle diverse and complex scenarios that typically cannot be accomplished by conventional LLM-basd methods.

Model Architecture

AgP operates through four essential modules. Planning module translates environmental descriptions into actionable plans, enabling the agent to execute specific actions and utilize tools as required. Skill library contains all the skills that are callable by the planning module through tool calling, ensuring interpretable and generalizable action execution for novel tasks. Memory module includes two types of memory, the long-term and short-term. The long-term memory stores records of successful actions and serves as an in-context example each time the planner is tasked with devising a new task. Conversely, short-term memory keeps track of changes in the world state to prevent incorrect actions by the robot. Execution module controls robots directly as the low-level control system, and further provides the execution feedback to the planning module for self-criticism and replanning. This feedback mechanism can help to avoid wrong actions and enhance planning validity.
